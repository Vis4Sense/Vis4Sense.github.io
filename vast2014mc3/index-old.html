Visual Analysis of Streaming Data with SAVI and SenseMAP

Project Description

Two tools were developed for the analysis tasks in the VAST Challenge 2014 Mini-Challenge 3: Social Analytics VIsualiszation (SAVI) and Sense Making with Analytic Provenance (SenseMAP).

SAVI provides linked timeline and geospatial view of the message streams. It allows search and filter based on text analysis, such as named entity extraction and sentiment analysis, and enable analyst to identity interesting events, tracking them temporally and geospatially, and find related information.

SenseMAP allows analysts to record finding, together with the visualisations that lead to it, and generate hypotheses and construct narratives. The entire sense making process is captured so it is possible to go back to any previous reasoning state and share or playback the process.

The fact that complex investigations are often carried out by several people working collaboratively is supported in our tools by allowing several people to use SAVI concurrently on the same dataset. Used in this mode, each users analyses are kept separate. However, the SenseMap tool supports collaboration and sharing of hypothesis and narrative structures. SenseMAP User Guide

NOTE: only support Chrome

User Registration

There is no authentication in this prototype. A new user simply enters their name and clicks Register to create a new one. If you're already registered, simply select your name on the list.

Sensemap Creation

Enter a name to create a new sensemap, such as "VAST Challenge 2014 MC1". An empty space will be displayed on the right side. If you want to open an existing sensemap, simply select it on the list.

Interface Overview

The left panel shows all findings that have been captured (by you or other users). The right panel shows the opening sensemap. The bottom panel shows the provenance of the opening sensemap (all past interactions).

Finding Creation

The atomic element of SenseMap is a finding. It can be created by clicking Add in the finding store. Fields in the dialog:
  • Image: you can copy an image and paste (Ctrl + V) to the dialog. Hint: use this Chrome extension to capture and edit screenshot. There's a function to copy image as well.
  • Note: a short thought of what you found
  • Description: a long description of what you found, can use to store extra information
  • Tags: separated by hitting tab, enter, or comma

Use a Finding in a Sensemap

Simply drag a finding from the left panel and drop it to the sensemap.

Interaction

  • Mouse wheel and drag to zoom and pan the whole sensemap
  • Freely move individual nodes (the findings dropped from the store) by mouse holding and dragging
  • Draw links between nodes: right mouse click onto the source node and drag to the target node.
  • View/Edit node by clicking it, the content will be shown in the finding store. Save/Cancel or simply clicking onto the empty space in the sensemap to close the content dialog.
  • Four menu buttons
    • Show full-screen captured image
    • Show provenance: nothing happens if the finding is captured in the server. Otherwise, it will open the visualization in the client when the finding was captured. Sound cool, isn't it? Please use the finding named "Finding from TimeSets" and see its provenance.
    • Collapse its children
    • Remove the node and its links
  • On the left side, above the finding store, there is an option to turn on/off the automatic layout. If on, the system will automatically arrange nodes and links.

NOTE

In this guide, finding and node refer to the same thing: a finding. But we use 'finding' when mentioning them on the left side - the finding store. And 'node' is used in the sensemap, together with links.

Provenance Panel

The bottom panel records all actions happened in the sensemap such as a new finding is added or a new link is drawn. Each action is shown using different symbols such as 'a' means 'a finding is added to the sensemap'. The user can click onto a circle to revert the sensemap to that state.

Findings Assembly from Multiple Sources [advanced]

Through the Add button mentioned above, you create a finding in the server. You can create findings from other sources, which are called clients. Each client needs to implement the capture feature and send to the server. At this status, there is no client yet, except the experimental TimeSets. You can try to zoom/pan that timeline visualization and click the Capture button at the bottom-right corner of the screen. Then, you should see a new finding in the store.

Asynchronous Collaboration

Different users capture findings themselves and the findings will be stored in the central place where others can see and use. The username is implicitly captured when he/see creates a new finding. So, we know who contributes what.

Synchronous Collaboration [advanced]

Please try scenario:
  1. Register one user, say user1
  2. Create a sensemap, say sensemap1. Add nodes and links to this.
  3. Open a new browser window, put it side-by-side with the previous one.
  4. In the new window, register another user, say user2. Then, click onto sensemap1 to open that sensemap using user2. You should see exactly as in the other window, the sensemap1.
  5. Play with sensemap1 on one window, you should see update immediately in another one. Try even add new finding or edit.
Summary: Two users open the same sensemap and collaborate in real-time.

Settings

Click onto the three-bar icon to open user/sensemap selection page.
